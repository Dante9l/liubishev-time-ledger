# 柳比歇夫时间账本插件需求文档

## 背景

用户需要一个 Obsidian 插件，用柳比歇夫计时法记录每日时间花销。重点不是持续计时，而是低摩擦的手动补录、稳定分类统计、可导出 Markdown，以及用 AI 生成周/月复盘。

## 目标

- 在 Obsidian 内快速补录时间段记录。
- 以固定分类加自由标签的方式沉淀稳定统计。
- 支持今日、本周、本月统计与复盘。
- 支持导出 Markdown 统计报告和原始流水。
- 接入可选 AI，生成周报/月报复盘，不影响离线基础功能。

## 交付物

- 一个可构建的 Obsidian 插件工程。
- 插件支持快速补录、记录列表、统计视图、Markdown 导出、AI 周/月复盘。
- 可运行的本地构建命令与基础验证。

## 约束

- 默认录入方式为开始时间 + 结束时间。
- 同时支持直接输入时长作为备用。
- 录入交互必须足够快，尽量在 10 秒内完成一条常规补录。
- 主数据保存在插件私有数据中，不以 Daily Note 为主存储。
- Daily Note/Markdown 仅作为展示与导出结果。
- AI 只读数据，不允许直接改写原始记录。
- 当前目录为空目录且非 Git 仓库，不依赖现有代码。

## 验收标准

- 能新增、编辑、删除时间记录。
- 能正确解析 `0910`、`09:10`、`0910-1040`、`90m` 等输入。
- 能按今日/周/月统计分类与标签。
- 能导出 Markdown 统计报告与 Markdown 原始流水。
- 能配置 AI Provider 并生成周报/月报复盘文本。
- `npm run build` 成功。

## 非目标

- 自动追踪电脑活动。
- AI 自动分类主流程。
- 复杂多端同步。
- 拖拽式时间轴编辑。

## 风险

- 时间输入解析存在歧义，需要明确优先级。
- 原始数据与展示导出需要保持一致。
- AI 请求失败时必须优雅降级，不影响本地功能。

## 假设

- 插件先以桌面端开发体验为主，但尽量保持 `isDesktopOnly: false`。
- 使用官方 sample plugin 结构作为脚手架基线。
